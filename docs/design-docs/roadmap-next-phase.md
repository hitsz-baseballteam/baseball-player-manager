# 下阶段迭代方案（修正版）

## 战略定位

聚焦 Web 端业务深度，以**"规则引擎 + 记录可靠性 + 裂变传播"**为主线。不做多端协同，不做过度抽象。

---

## Sprint 1：记录可靠性 + 裂变（当前冲刺，约 1 周）

### [ ] Undo 撤销机制

**痛点**：现场记录手误频率高，一旦点错打击结果无法回退。

**方案**：
- 在 `ScoreboardGame` 的 `LiveInning` 中维护一个 `eventLog: PlateAppearance[]` 队列
- `undo()` 从队列末端弹出最后一个 PA，基于前序 PA 重算整个 `statLines` 和 `runners`
- UI 上在控制栏加一个「↩ 撤销」按钮，最近一次操作高亮提示
- 仅支持撤销最近 1 步（MVP），后续可扩展为多步撤销

**不做**：复杂的命令模式或 Redux 式时间旅行。事件队列 + 重算足够覆盖当前需求。

**工时**：2 天

---

### [ ] 战报长图一键生成

**痛点**：队长赛后要手动修图写文案发推送，耗时且质量不稳定。

**方案**：
- 引入 `html2canvas`，在赛后 REVIEW 阶段增加「生成战报」按钮
- 渲染两张固定模板：
  - **Box Score 战报** (16:9，公众号横版)：比赛概要 + 每局得分表 + 打者数据表 + 投手数据表 + 守备亮点
  - **MVP 球员卡片** (9:16，小红书竖版)：今日 MVP 球员头像位 + 单场数据亮点 + 队徽水印
- 模板复用现有 CSS 变量和组件样式，在一个隐藏 DOM 节点中渲染后截图下载

**不做**：复杂的模板编辑器、多套皮肤。先给一套默认模板，够用即可。

**工时**：3 天

## Sprint 2：数据可视化 + 赛季抽象（下个版本，约 2 周）

### [ ] 球员打击喷雾图 (简化版)

**痛点**：数据中心现在只有数字表格，缺乏直观的可视化。

**方案**：
- **不做落点精确采集**。打击结果自动分类：
  - 1B/2B → 中外野/右外野方向（推打）
  - 3B/HR → 左外野/中外野方向（拉打）
  - GO → 内野左/中/右（按防守位置分配）
  - FO → 外野左/中/右
- 在球员数据页渲染一个简化的棒球场 SVG 俯视图，按方向分类展示打击分布
- 这不是真正的 Spray Chart，但能给教练足够的决策信息（"这个打者推打多" vs "拉打多"）

**不做**：精确落点坐标采集。大学比赛记录员没有这个精力。

**工时**：3 天

---

### [ ] 赛季层级

**痛点**：目前靠 `game.date.substring(0,4)` 推导赛季，无法处理跨年赛季（如"2025秋季赛季"）。

**方案**：
- 在 Workspace 的 `preferences` 或顶层加一个轻量 `seasons` 数组：
  ```ts
  seasons: [{ id, name, startDate, endDate }]
  ```
- `Game` 加一个可选字段 `seasonId`
- 数据中心赛季下拉框改为读取 `seasons` 列表
- 名人堂的 `computeSeasonBadges` 改用 `seasonId` 分组
- 向后兼容：没有 `seasonId` 的旧比赛仍按年份展示

**不做**：独立的 `Season` 表/实体、多球队/多联盟抽象。当前一所学校的单一球队不需要这些。

**工时**：4 天（含迁移旧数据）

### [ ] 新人开荒进度条

**不做**：全自动考核判定（需要额外的训练记录模块）。MVP 手动勾选即可。
