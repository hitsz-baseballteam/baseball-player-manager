# Archive

This directory holds documentation that is no longer part of the main development path.

Use the archive for:

- point-in-time reviews and scorecards
- superseded execution plans
- one-off QA handoff artifacts

Do not use the archive as the primary source of current behavior. For active system facts, start from:

- `docs/ARCHITECTURE.md`
- `docs/API.md`
- `docs/SCHEMA.md`
- `docs/SECURITY.md`
- `docs/RELIABILITY.md`
- `docs/FRONTEND.md`
- `docs/DESIGN.md`
- `docs/PRODUCT_SENSE.md`

Archive layout:

- `assessments/` — dated review reports and quality score snapshots
- `design/` — one-off design QA handoff notes
- `design-qa-assets/` — screenshots and comparison assets tied to archived design QA
- `../exec-plans/archive/` — superseded plans and review memos that no longer guide active work
