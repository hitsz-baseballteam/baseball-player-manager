# 20260605 排阵页布局重排

## 完成状态
- 状态：已完成
- 完成日期：2026-06-05
- 实际验证：`npm test` ✅ / `npm run lint` ✅ / `npm run build` ✅ / 浏览器验收 ✅
- 浏览器验收截图：`/tmp/lineup-reflow-desktop-full.png`、`/tmp/lineup-reflow-medium-full.png`

## 目标
修正 `/lineup` 页面在中等宽度下两侧面板过宽、挤压中间守位图的问题，把排阵页重排为“球场图在左，右侧上下堆叠”的响应式工作台，并让左侧球员区成为独立滚动面板，而不是把页面整体拉长。

## 已确认决策

| 决策 | 选择 |
|---|---|
| 中等宽度重排结构 | **B：球场图在左，右侧上下堆叠** |
| 左侧全队球员区 | 独立滚动，不再通过整页纵向拉长承载长名单 |
| 右侧打线顺序 | 保持 9 个固定槽位，不做结构性调整 |

## 当前问题（已核实）
- `src/components/lineup-page-client.module.css` 当前使用固定三列：`280px 1fr 260px`
- 三列只在 `max-width: 900px` 时才切为单列，断点过晚
- 左栏球员卡列表过长，导致球场图失去主视觉优先级
- 当前布局优先级错误：辅助面板比核心守位图更占空间

## 范围
### 要做
- 调整 `/lineup` 主布局断点与网格结构
- 让桌面宽屏仍可用三栏，但中间守位图优先级更高
- 在中等宽度切换为：左侧大球场、右侧“打线顺序 + 全队球员”上下堆叠
- 让球员区成为可滚动区域，并保证不会把整页无限拉长
- 视需要微调 `BenchPanel` / `FieldBoard` / `LineupOrder` 的容器高度、overflow、边框方向与间距
- 补充/更新布局测试（至少锁定主要 DOM 结构和关键文案）
- 浏览器复验 `/lineup` 页面
- 更新相关文档与 `docs/QUALITY_SCORE.md`（如测试数变化）

### 不做
- 不改排阵业务逻辑（assign/swap/move/save）
- 不重写拖拽协议
- 不改右侧打线 9 槽位的交互模型
- 不对视觉语言做整页重设计，只做布局重排与必要 polish

## 实施步骤
1. **复盘现状布局边界**
   - 读取 `lineup-page-client.module.css`、`bench-panel.module.css`、`field-board.module.css`、`lineup-order.module.css`
   - 明确现有 `overflow` / `height` / `border` / `grid` 约束
2. **重排主布局**
   - 为 `.board` 设计新的宽屏 / 中宽 / 窄屏断点
   - 中宽采用“左球场、右侧上下堆叠”
   - 必要时在 `lineup-page-client.tsx` 内增加中宽容器分组结构
3. **收口滚动行为**
   - 左侧（中宽下右下区域中的球员面板）改为独立滚动区
   - 保证右侧打线区固定可见、球员区在剩余空间内滚动
   - 避免整页因球员卡列表被拉得过长
4. **微调子面板样式**
   - `BenchPanel`：滚动区、标题、卡片节奏、边框方向
   - `FieldBoard`：最大宽度、居中方式、最小高度
   - `LineupOrder`：固定 9 槽位下的高度与边界
5. **验证**
   - 运行测试 / lint / build
   - 浏览器打开 `/lineup` 做桌面与中等宽度验收
6. **文档更新**
   - 更新 `docs/QUALITY_SCORE.md`
   - 如需要，记录到相关已完成计划或前端文档

## 风险
| 风险 | 影响 | 缓解 |
|---|---|---|
| 只改 CSS 无法完成中宽重排 | 需要更改 DOM 结构 | 允许在 `lineup-page-client.tsx` 增加布局包裹层 |
| 子面板各自 `overflow` 冲突 | 滚动条错位、内容被裁切 | 逐层核对父容器高度/最小高度/overflow |
| 新断点影响拖拽命中区域 | 交互回归 | 浏览器复验拖拽/点击入口，必要时补测 |
| 右侧上下堆叠后高度分配不均 | 打线或球员区被挤压 | 先固定打线区高度，再让球员区吃剩余空间 |
