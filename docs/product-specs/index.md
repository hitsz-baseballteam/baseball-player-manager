# 产品规格

功能规格文档，供 agent 实施时参考。每个规格描述：用户需求、验收标准、边界情况、约束条件。

## 格式

```markdown
# SPEC-NNN: 功能名称

状态：[提议 | 开发中 | 已完成]

## 用户需求
[用户想完成什么，为什么这个功能重要]

## 验收标准
- [ ] [可验证的条件 1]
- [ ] [可验证的条件 2]

## 边界情况
- [特殊情况 1]：[预期行为]
- [特殊情况 2]：[预期行为]

## 约束
- [技术约束 1]
- [产品约束 2]
```
